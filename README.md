# Read Aloud 英語口說跟讀練習器

一個免安裝、免金鑰、單一 HTML 檔的英語口說練習工具。載入課文或影片字幕後，學生對著麥克風朗讀，系統即時辨識、比對、回饋，練完提供錄音回放與卡關發音學習卡。專為國小英語課堂設計。

## 功能特色

- **兩種練習模式**
  - 🎯 **逐詞挑戰**：唸對的詞變綠、螢光筆高亮逐詞前進，唸錯達次數自動示範發音，練準確度
  - 🗣 **影子跟讀**：一句一回合「聽整句示範 → 跟著唸 → 句子級評分」，長句自動切成 5~12 詞的跟讀單位，練語流與節奏
- **四種文本來源**：TXT 純文字、SRT 字幕檔、Language Reactor 匯出的 XLSX（含中文翻譯對照）、YouTube「顯示轉錄稿」貼上匯入（時間戳自動去除）
- **自動併句**：字幕斷行自動重組為完整句子，中文翻譯跟隨對應句子顯示
- **字數關卡制**：長文自動切關（預設每關約 150 字、於句尾分段），關卡列顯示星等、可跳關重練
- **練習結果**：完成字數鼓勵語、本關錄音回放與下載、卡關發音學習卡（依卡關次數排序，可展開常用片語跟唸）
- **進度記憶**：關卡進度與星等自動儲存於瀏覽器，下次開啟同一篇文章接續練習

## 容錯與比對機制

針對真實口說與辨識引擎的特性，內建多道容錯防線，減少「明明唸對卻判錯」的挫折：

| 機制 | 處理的情況 |
|---|---|
| 相似度比對（可調 60%~100%） | 小拼字差異：elephant / elephants |
| 文字正規化 | 大小寫、標點、縮寫（I'm = I am）、千分位 |
| 數字格式 | 序數（21st）、西元年（1984）、金額（$5）、百分比（50%）、千位數 |
| 同音詞容錯 | to / two / too、their / there 一律算對 |
| 功能詞變體 | the 強讀被轉成 thee/duh、an 被轉成 and 等引擎輸出 |
| 冠詞順帶通過 | 引擎省略 a / an / the 時自動帶過；示範發音改為連同下一詞連唸 |
| 縮讀雙向對映 | gonna ↔ going to、wanna ↔ want to、Mr. ↔ mister、OK ↔ okay |
| 語助詞忽略 | um、uh、hmm 等思考聲不記錯、不干擾進度 |
| 回讀忽略 | 重唸剛唸過的詞、整句重來（self-repair）不記錯 |
| 合併比對 | 引擎把兩詞併成一個（ice + cream）也能對上 |

## 部署到 GitHub Pages

1. 在 GitHub 建立新儲存庫（Repository），例如 `read-aloud`，設為 **Public**
2. 將 HTML 檔**改名為 `index.html`** 後上傳（Add file → Upload files）——網址乾淨且避免中文檔名編碼
3. 進入 **Settings → Pages**，Source 選 `Deploy from a branch`，Branch 選 `main`、資料夾 `/ (root)`，按 Save
4. 等待 1~2 分鐘，頁面上方出現網站網址，即可分享給學生

**更新版本**：上傳同名檔案覆蓋，Commit 後約 1 分鐘生效。學生看到舊版時按 `Ctrl + F5` 強制重新整理。

**本機使用**：下載 HTML 檔直接用 Chrome 開啟即可，功能相同（進度記憶綁定該台電腦）。

## 使用說明

### 載入文本

| 來源 | 操作方式 | 中文翻譯 | 適用情境 |
|---|---|---|---|
| TXT | 載入 TXT / SRT / XLSX | 無 | 課本課文、自編教材 |
| SRT | 載入 TXT / SRT / XLSX | 無 | 影片字幕檔 |
| XLSX | 載入 TXT / SRT / XLSX | **有**（C 欄） | Language Reactor 匯出（B 欄英文、C 欄翻譯） |
| YouTube 逐字稿 | 貼上逐字稿 | 無 | 影片頁「顯示轉錄稿」→ 全選複製 → 貼上 |

> 提醒：YouTube 純自動產生的字幕常缺少標點，「自動併句」會無從斷句；此時建議關閉自動併句改用原始斷行。

### 逐詞挑戰模式

1. 按「▶ 開始練習」並允許麥克風權限
2. 唸出**黃色螢光筆**標記的詞，唸對自動前進（可連續順唸）
3. 唸錯達設定次數會閃紅並自動示範（冠詞會連同下一個詞一起示範），卡住可「⏭ 跳過此詞」
4. 點擊課文任何一個詞可跳到該處開始練

### 影子跟讀模式

1. 切換到「🗣 影子跟讀」後按開始
2. 系統唸出目前跟讀單位（黃框標示）→「換你唸這一句！」跟著唸
3. 唸完或停頓 4 秒結算：句中唸對詞數比例達「容許彈性」門檻即過，自動進下一句
4. 未達標自動重播示範再試；達嘗試上限帶過，漏詞（粉紅標示）轉入卡關學習卡
5. 「🔊 再聽這句」重播示範不計次數；「⏭ 跳過此句」跳過整句；點擊任一詞跳到該句

### 每關結束

結果卡顯示：鼓勵語、一次唸對／聽了示範／跳過統計、🎧 本關錄音回放（可下載）、🎯 卡關發音學習卡（點卡片展開該詞的常用片語，每個片語可點 🔊 跟唸）。按「下一關 ▶」接續；全部關卡完成顯示 🏆 全篇成就。

### 設定選項

| 選項 | 預設 | 說明 |
|---|---|---|
| 比對容許彈性 | 標準 80% | 逐詞模式的單詞相似度門檻；也是影子模式的句子通過比例 |
| 唸錯幾次後示範發音 | 2 次 | 逐詞模式的示範觸發次數；也是影子模式的重試上限 |
| 示範發音語速 | 0.8x | 國小建議 0.7x~0.8x |
| 每關練習字數 | 150 字 | 60~300 字，於句尾分關；調整後重新分關並清除本篇星等 |
| 自動併句 | 依來源 | XLSX / SRT / 逐字稿預設開啟，TXT 預設關閉 |
| 辨識口音 | en-US | 依教材口音選擇美式／英式／澳洲 |

## 系統需求與限制

- **瀏覽器**：電腦版 **Chrome 或 Edge**，或 **Android 平板的 Chrome**
- **iPad / iPhone 不支援**：iOS 上所有瀏覽器（含 Chrome）都被要求使用 WebKit 引擎，未提供本工具所需的連續語音辨識；請改用電腦或 Android 裝置
- **網路**：語音辨識與片語查詢需連網（語音由瀏覽器送至 Google 伺服器辨識）
- **麥克風**：首次使用需允許權限；多人同時練習建議戴耳麥或拉開間距，避免互相收音
- **辨識準確度**：對兒童聲音與非母語口音辨識率會打折——這正是容錯防線存在的原因，可視班級狀況調寬彈性
- **評分顆粒度**：只能判斷「詞有沒有唸出來」，無法評重音、語調與音素層級的發音品質；流暢度與語感請搭配錄音回放與教師回饋

## 資料與隱私

- 語音辨識由瀏覽器內建服務處理（送至 Google 伺服器），本工具**不會**將語音上傳到任何自架伺服器
- 練習錄音只存在使用者的瀏覽器記憶體中，關閉頁面即消失，除非自行下載
- 進度與星等以 localStorage 存在**該台電腦的該瀏覽器**中，不跨裝置同步；無痕模式不保存
- 學習卡片語查詢使用免費的 [Datamuse API](https://www.datamuse.com/api/)，僅送出查詢的英文單字

## 常見問題

**Q：按開始後麥克風沒反應？**
點網址列左側的鎖頭／設定圖示，將麥克風權限改為「允許」後重新開始。本機開啟 HTML 若被擋，改用 GitHub Pages 網址。

**Q：明明唸對了卻判錯？**
調寬「比對容許彈性」（例如 70%）；確認辨識口音與教材相符；降低環境噪音。

**Q：唸完到判定之間有一點延遲？**
這是刻意設計：辨識引擎的「暫時結果」會跳動修正，系統只用它做快速推進，所有記錯與略過的判斷都等「最終結果」才執行——用少許延遲換取不誤判。

**Q：影子模式的示範播到一半沒聲音或卡住？**
已內建看門狗與防中斷保護，逾時會自動進入跟唸階段。若仍發生請回報該句內容。

**Q：某個詞怎麼唸都過不了？**
專有名詞偶爾辨識不出來，按跳過即可，它會列入卡關學習卡供課後加強。

**Q：併句之後翻譯的位置怪怪的？**
機器翻譯照原始斷行切分，重新併句後句界附近會略有位移，屬資料先天限制。

**Q：學生換電腦後進度不見了？**
進度存在原本那台電腦的瀏覽器中，屬預期行為。
